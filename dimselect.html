Principle Component Analysis

Principle component analysis, or PCA is a technique that is widely used for applications such as dimensionality reduction. PCA can be defined as the orthogonal projection of the data onto a lower dimensional linear space, know as the principle subspace, such that the variance of the projected data is maximized.

Solution

The variance will be a maximum when we set \(\mathbf{u}_1\) equal to the eigenvector having the largest eigenvalue \(\lambda_1\). This eigenvector is known as the first principle component.

Here, the eigenvalue measures the relative importance of each principle component. Dimension reduction is then achieved by selecting only the top few coordinates (principle components).

The problem that we shall focus on now is that of deciding how many coordinates should be retained.

Dimensionality Selection

The gist of our method consists of explicitly constructing a model for the numbers \(\lambda_j\) \((j = 1,...,M)\) and estimating the position of the “gap” or the “elbow” by maximizing a profile likelihood function.

Scree plot

Plot the eigenvalues in descending order (often called a scree plot) and look for a “big gap” or an “elbow” in such a graph.

# Assumes distribution of eigenvalues follow uniform distribution
d1=runif(10,0,10) # First 10 eigenvalues
d2=runif(10,15,25) # Last 10 eigenvalues
d = c(d1,d2)
d = sort(d,decreasing = TRUE)
barplot(
    height = d,
    ylim = c(0,25),
    xlab = "Principle Component", ylab = "Variance", main = "Screeplot")

The Method

Again, let \(\lambda_1\geq\lambda_2\geq...\geq\lambda_M>0\) be the ordered measures of importance of the corresponding coordinates. In the case of PCA these are ordered eigenvalues. Our basic idea is very simple. Given a fixed number \(1\leq N\leq M\), write \({\mathcal{S}}_1=\{\lambda_1,\lambda_2,...,\lambda_N\}\) and \({\mathcal{S}}_2=\{\lambda_{N+1},\lambda_{N+2},...,\lambda_M\}\). If a “gap” or an “elbow” exists at position \(N\), then we can think of \({\mathcal{S}}_1\) and \({\mathcal{S}}_2\) as samples from two different distributions. Based on this point of view, we proceed to model \({\mathcal{S}}_1\) as an independent sample from \(f(\lambda|\theta_1)\) and \({\mathcal{S}}_2\) as an independent sample from \(f(\lambda|\theta_2)\).

The log-likelihood function under the naïve independence assumption can be written as \[ \begin{aligned} l(N,\theta_1,\theta_2)=\sum_{i=1}^{N}{\text{log}f(\lambda_i|\theta_1)}+\sum_{j=N+1}^{M}{\text{log}f(\lambda_j|\theta_2)}. \end{aligned} \]

For any given \(N\), MLEs of \(\theta_1\) and \(\theta_2\) can be obtained separately from \({\mathcal{S}}_1\) and \({\mathcal{S}}_2\). By plugging in these estimates, we obtain a profile log-likelihood for \(N\):

\[ \begin{aligned} l_q(q)=\sum_{i=1}^{N}{\text{log}f(\lambda_i|\hat{\theta}_1(q))}+\sum_{j=N+1}^{M}{\text{log}f(\lambda_j|\hat{\theta}_2(q))} \end{aligned} \]

We use the notation \(\hat{\theta}_j(q)\) \((j=1,2)\) to emphasize that the MLE for \(\theta_1\) and \(\theta_2\) may depend on \(N\). An estimate of \(N\) can then be obtained by maximizing the profile log-likelihood above:

\[ \begin{aligned} \hat{q} = \underset{k=1,2,...,M}{\text{argmax}}l_q(k) \end{aligned} \]

For simplicity, we choose \(f\) to be the Gaussian Distribution:

\[ \begin{aligned} f(\lambda|\theta)=f(\lambda|\mu_j,\sigma^2)=\frac{1}{\sqrt{2\pi\sigma^2}}\text{exp}\{-\frac{(d-\mu_j)^2}{2\sigma^2}\} \quad \text{for j=1,2} \end{aligned} \]

Note here that it is important to use a common scale parameter \(\sigma\) for both \({\mathcal{S}}_1\) and \({\mathcal{S}}_2\). If a different \(\sigma\) is used for each model, the model becomes too flexible and it is possible for the profile log-likelihood to become infinite. Here, the MLEs for \(\mu_1\), \(\mu_2\), \(\sigma^2\) are: \[ \begin{equation*} \begin{aligned} & \hat{\mu}_1 = \frac{\sum_{\lambda_i\in{\mathcal{S}}_1}{\lambda_i}}{N} \\ & \hat{\mu}_2 = \frac{\sum_{\lambda_j\in{\mathcal{S}}_2}{\lambda_i}}{M-N} \\ & \hat{\sigma}^2=\frac{(N-1)s_1^2+(M-N-1)s_2^2}{M-2} \end{aligned} \end{equation*} \]

where \(s_j^2\) is the sample variance of \({\mathcal{S}_j}\)

Testing/Simulated Examples

## Loading required package: graphstats
## Loading required package: pracma
## Loading required package: eigeninv
elbows = sort(sample(40,2)) + 5
while (elbows[2] - elbows[1] < 10) {
  elbows = sort(sample(40,2)) + 5
}
eval_1 = sort(runif(elbows[1], min = 50, max = 60),decreasing = TRUE)
eval_2 = sort(runif(elbows[2]-elbows[1], min = 30, max = 40),decreasing = TRUE)
eval_3 = sort(runif(50 - elbows[2], min = 10, max = 20),decreasing = TRUE)
eval = c(eval_1,eval_2,eval_3)
S = eigeninv::eiginv(eval,50,symmetric=TRUE) # Generate covariance matrix based on eigenvalues
X = randn(1000, 50);
for (i in 1:50) {
  m = mean(X[,i])
  X[,i] = X[,i] - m
}
X = X %*% inv(chol(cov(X)));
X = X %*% chol(S);
pca = princomp(X)
dim = dimselect(pca$sdev,n = 2)
print(dim)
## [1] 26 41
print(elbows)
## [1] 25 41
screeplot(pca,npcs = 50)

barplot(eval)
